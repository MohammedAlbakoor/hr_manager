# نشر المشروع مجانًا

هذا المشروع مكوّن من جزأين:

- واجهة Flutter Web
- باك إند Laravel API

أفضل مسار مجاني حاليًا للمبتدئ:

- الواجهة: Cloudflare Pages
- الباك إند: InfinityFree أو Render/Koyeb حسب مدى تقبلك للقيود

## التوصية السريعة

إذا كان همّك الأساسي أن يصبح المشروع أونلاين بأقل تعقيد:

1. انشر الواجهة على Cloudflare Pages.
2. انشر الباك إند على استضافة PHP مجانية.
3. ابنِ الواجهة باستخدام رابط الـ API الحقيقي.

## ما تم التحقق منه محليًا

- بناء الويب نجح: `flutter build web --release`
- اختبارات Laravel نجحت
- مجلد `build/web` الحالي جاهز للرفع المباشر
- عدد ملفات الواجهة الحالية: `36`
- الحجم الإجمالي التقريبي: `38 MB`

هذا يعني أن رفع الواجهة إلى Cloudflare Pages بالسحب والإفلات ممكن مباشرة.

## الخطوة 1: نشر الواجهة على Cloudflare Pages

### بناء الواجهة

من جذر المشروع شغّل:

```powershell
flutter build web --release --dart-define=API_BASE_URL=https://YOUR_API_DOMAIN/api
```

استبدل:

- `https://YOUR_API_DOMAIN/api`

برابط الباك إند الحقيقي بعد نشره.

### الرفع

بعد إنشاء حساب Cloudflare:

1. افتح `Workers & Pages`
2. اختر `Create application`
3. اختر `Pages`
4. اختر `Direct Upload`
5. ارفع محتويات المجلد:

```text
build/web
```

### ملاحظة مهمة

هذا المشروع لا يستخدم `PathUrlStrategy` حاليًا، لذلك روابط Flutter Web ستكون بنمط hash بشكل افتراضي، وهذا يخفف مشاكل إعادة التوجيه على الاستضافة.

## الخطوة 2: نشر الباك إند

### خيار A: InfinityFree

هذا الخيار مجاني بالكامل، لكنه أصعب مع Laravel.

قبل الرفع جهّز ملف البيئة:

1. انسخ:

```text
backend/.env.example
```

إلى:

```text
backend/.env
```

2. غيّر القيم الأساسية:

```dotenv
APP_ENV=production
APP_DEBUG=false
APP_URL=https://YOUR_API_DOMAIN
CORS_ALLOWED_ORIGINS=https://YOUR_FRONTEND_DOMAIN
DB_CONNECTION=mysql
DB_HOST=YOUR_DB_HOST
DB_PORT=3306
DB_DATABASE=YOUR_DB_NAME
DB_USERNAME=YOUR_DB_USER
DB_PASSWORD=YOUR_DB_PASSWORD
```

3. ولّد مفتاح التطبيق محليًا:

```powershell
cd backend
php artisan key:generate --show
```

ثم ضعه داخل:

```dotenv
APP_KEY=base64:...
```

### ملفات Laravel التي سترفعها

ارفع مجلد `backend` بعد استبعاد ما لا يلزم مثل:

- `tests`
- `storage/logs`
- `.env` المحلي إذا كان يحتوي بيانات غير إنتاجية

### ملاحظات مهمة جدًا عن InfinityFree

- Laravel يعمل عليه بصعوبة وليس بسلاسة كاملة.
- لا يوجد وصول CLI/Artisan على الخادم المجاني.
- لذلك إعداد قاعدة البيانات والنشر يحتاجان عناية إضافية.

إذا اخترت هذا المسار، الأفضل أن أجهز لك في الخطوة التالية نسخة مخصصة لـ InfinityFree داخل المشروع.

### خيار B: Render أو Koyeb

هذا المسار أسهل تقنيًا مع Laravel، لكنه ليس "مجانيًا مثاليًا":

- Render المجاني ينام بعد الخمول
- Koyeb المجاني لديه قيود على قاعدة البيانات ويتطلب تحقق وسيلة دفع

إذا اخترت هذا المسار، سأجهز لك ملفات Docker والنشر المناسبة.

## الخطوة 3: ربط الواجهة بالباك إند

بعد أن يصبح للباك إند رابط حقيقي:

1. أعد بناء الواجهة بالأمر:

```powershell
flutter build web --release --dart-define=API_BASE_URL=https://YOUR_API_DOMAIN/api
```

2. أعد رفع `build/web` إلى Cloudflare Pages

## الخطوة 4: التحقق بعد النشر

تحقق من:

1. فتح صفحة تسجيل الدخول
2. نجاح تسجيل الدخول
3. ظهور البيانات في لوحة الموظف أو المدير
4. عمل صفحة الإشعارات
5. عمل QR
6. عمل إضافة موظف وتعديل كلمة المرور

## ماذا أستطيع أن أفعل أنا؟

أستطيع:

- تجهيز ملفات النشر داخل المشروع
- تجهيز إعدادات الإنتاج
- تجهيز نسخة مناسبة للاستضافة التي تختارها
- فحص البناء والاختبارات
- إعطاؤك الخطوات الدقيقة حسب الاستضافة

لا أستطيع وحدي:

- إنشاء حسابات خارجية بدلًا عنك
- حل captcha أو تسجيل الدخول لحسابات الاستضافة نيابة عنك
- إتمام الرفع إلى لوحة خارجية بدون حساب أو وصول منك

## القرار الأفضل الآن

إذا أردت أقل مخاطرة وأبسط تجربة:

- الواجهة: Cloudflare Pages
- الباك إند: أحتاج منك أن تختار بين:
  - `InfinityFree` إذا أردت مجاني 100%
  - `Render` إذا قبلت خدمة تنام بعد الخمول
  - `Koyeb` إذا قبلت التحقق ببطاقة مع بقاء الخدمة على free tier
